# Contributing

Contributions are welcome. This project favours **small, well-tested, deterministic**
changes.

## Setup

```bash
python -m venv .venv
source .venv/bin/activate        # Windows: .venv\Scripts\activate
pip install -e ".[dev]"
export CV_API_KEY=dev-key CV_LLM_MODE=mock
```

## Before you open a pull request

Everything runs in **mock mode** — no models, no network:

```bash
ruff check .            # lint
ruff format --check .   # formatting
mypy app                # types (must stay clean)
pytest                  # 130 tests, deterministic
```

CI runs exactly these on Python 3.11 and 3.12, plus a secret scan and a
dependency audit. Please keep all of them green.

## Guidelines

- **Mock-first.** Any model-dependent behaviour must have a deterministic mock
  path so the test suite stays network-free. Model calls funnel through
  `app/pipeline/llm.py` and `app/query/embeddings.py`.
- **Trust boundary.** Facts that must be *true* (contact details, years, skill
  presence, page numbers, scores) are computed in code, never trusted from the
  model. New model output must be verified before it is surfaced or scored.
- **No PII in logs.** Use `app/logging_safe.py`; never log candidate content.
- **Tests over count.** Add tests for real behaviour and edge cases, not to pad a
  number. See [docs/evaluation.md](docs/evaluation.md).
- **Docs.** Update the relevant file in [docs/](docs/) when you change behaviour;
  record notable design choices in [docs/design-decisions.md](docs/design-decisions.md).

## Synthetic data only

Never commit real candidate data or any real organisational identity. The
corpus is generated by `scripts/generate_synthetic_data.py` and is git-ignored.
